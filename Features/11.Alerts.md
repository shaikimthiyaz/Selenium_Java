# Alerts

Alerts are **pop‑up dialogs** generated by the **browser**(not HTML elements).

### Types of alerts:

**1.Simple Alert** → shows a message with an **OK** button.

**2.Confirmation Alert** → shows a message with **OK and Cancel** buttons.

**3.Prompt Alert** → allows user input with a **text box plus OK/Cancel**.

## Handling Alerts in Selenium Java

Selenium provides the **`Alert`** **interface** to interact with alerts.

### 1. Switch to Alert

```
Alert alert = driver.switchTo().alert();
```

### Handle Simple Alert

```
Alert simpleAlert = driver.switchTo().alert();
System.out.println(simpleAlert.getText());  // read message
simpleAlert.accept();                       // click OK
```

### Handle Confirmation Alert

```
Alert confirmAlert = driver.switchTo().alert();
System.out.println(confirmAlert.getText());
confirmAlert.dismiss();   // click Cancel
// or confirmAlert.accept(); // click OK
```

### Handle Prompt Alert

```
Alert promptAlert = driver.switchTo().alert();
System.out.println(promptAlert.getText());
promptAlert.sendKeys("Selenium Java");  // enter text
promptAlert.accept();                   // click OK
```

## interview questions

### 1.Why switchTo()? 

Alerts are outside the DOM, so you must switch driver’s focus using  `driver.switchTo().alert()`.

### 2.What happens if no alert is present?

Selenium throws `NoAlertPresentException`.

### 3. How to synchronize with alerts? 

Use explicit waits:
```
WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
wait.until(ExpectedConditions.alertIsPresent());
Alert alert = driver.switchTo().alert();
```

### 4.Difference between alerts and popups?

Alerts → browser‑native, handled via Alert interface.

Popups → HTML elements (like modal dialogs), handled via locators.
