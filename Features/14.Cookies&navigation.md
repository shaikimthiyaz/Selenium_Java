##  Cookie Operations in Selenium

| Operation                | Code Example                                                   | Real-World Use Case                                      |
|--------------------------|---------------------------------------------------------------|----------------------------------------------------------|
| **Add a cookie**         | `driver.manage().addCookie(new Cookie("username","testUser"));` | Simulate a logged-in session by injecting a session cookie. |
| **Get all cookies**      | `Set<Cookie> cookies = driver.manage().getCookies();`           | Validate that cookies are created after login (e.g., session, preferences). |
| **Get a specific cookie**| `Cookie c = driver.manage().getCookieNamed("username");`        | Fetch a particular cookie to check its value (e.g., session ID). |
| **Delete a specific cookie** | `driver.manage().deleteCookieNamed("username");`            | Remove a cookie to simulate logout or reset user state. |
| **Delete all cookies**   | `driver.manage().deleteAllCookies();`                          | Clear all cookies before starting a new test to avoid session conflicts. |


##  Navigation Methods in Selenium

| Method      | Syntax                                           | Real-World Use Case                                      |
|-------------|--------------------------------------------------|----------------------------------------------------------|
| **to()**    | `driver.navigate().to("https://example.com");`   | Navigate to a new URL (similar to `driver.get()`).       |
| **back()**  | `driver.navigate().back();`                      | Go back to the previous page (like browser back button). |
| **forward()** | `driver.navigate().forward();`                 | Move forward to the next page (like browser forward button). |
| **refresh()** | `driver.navigate().refresh();`                 | Reload the current page (useful for dynamic content).    |

##  Difference Between `driver.get()` and `driver.navigate().to()`

| Feature                  | `driver.get()`                                | `driver.navigate().to()`                          |
|---------------------------|-----------------------------------------------|--------------------------------------------------|
| **Purpose**               | Opens a URL directly.                         | Opens a URL as part of the navigation interface. |
| **Return Type**           | Void (no return value).                       | Void (no return value).                          |
| **Browser History**       | Does not provide navigation controls.         | Supports chaining with back, forward, refresh.   |
| **Usage Simplicity**      | Simple and straightforward for loading pages. | More flexible, used when simulating browser-like behavior. |
| **Common Use Case**       | Initial page load in tests.                   | Moving between pages, testing back/forward flows. |

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;

public class NavigationExample {
    public static void main(String[] args) {
        // Launch browser
        WebDriver driver = new ChromeDriver();

        // 1. Open Google using get()
        driver.get("https://www.google.com");
        System.out.println("Opened Google using get()");

        // 2. Navigate to Facebook using navigate().to()
        driver.navigate().to("https://www.facebook.com");
        System.out.println("Navigated to Facebook using navigate().to()");

        // 3. Go back to Google
        driver.navigate().back();
        System.out.println("Navigated back to Google");

        // 4. Move forward to Facebook
        driver.navigate().forward();
        System.out.println("Navigated forward to Facebook");

        // 5. Refresh the current page
        driver.navigate().refresh();
        System.out.println("Refreshed Facebook page");

        // Close browser
        driver.quit();
    }
}
