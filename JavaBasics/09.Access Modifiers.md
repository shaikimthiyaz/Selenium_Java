# Access modifiers

## What Are Access Modifiers?

Access modifiers define the **visibility and accessibility** of classes, methods, and variables.

Java has four access levels:

private

default (no keyword)

protected

public

## Why Access Modifiers Are Needed (Real Problems)

Without access control:

Anyone can misuse framework code

Test stability breaks

Hard to maintain large codebases

Access modifiers help:
✔ Encapsulation

✔ Security

✔ Maintainability

✔ Clear API boundaries



| Modifier  | Same Class | Same Package | Subclass (diff pkg) | Anywhere |
| --------- | ---------- | ------------ | ------------------- | -------- |
| private   | ✔          | ❌            | ❌                   | ❌   |
| default   | ✔          | ✔            | ❌                   | ❌    |
| protected | ✔          | ✔            | ✔                   | ❌     |
| public    | ✔          | ✔            | ✔                   | ✔      |


## private Access Modifier

Accessible only **within the same class**

```
class LoginPage {
    private WebDriver driver;

    private void enterPassword() {
        // hidden internal logic
    }
}
```

✔ Prevents misuse
✔ Enforces encapsulation

## Default

Accessible only within the same package & same class

```
class BaseTest {
    void setup() {
        // accessible only within same package
    }
}
```

Used for internal framework logic

## protected

Accessible within:

Same package

Subclasses in other packages

```
public class BasePage {
    protected WebDriver driver;

    protected void click(WebElement element) { ... }
}
```
```
public class LoginPage extends BasePage {
    public LoginPage(WebDriver driver) {
        this.driver = driver; // allowed because protected
    }
}
```


All page objects (subclasses) need access to driver

But tests should NOT directly access driver

And classes in the same package can also use it

Protected = inheritance-friendly encapsulation.

## Public

Accessible from anywhere
