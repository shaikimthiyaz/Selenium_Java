### Imagine you buy a universal remote control.

This remote has buttons like:

> power on/off

> increase volume

> change channel

But the remote does not know how each TV works internally.

ðŸ‘‰ The remote only defines what buttons exist.

This is exactly what an interface does.

### Java Interface Example (Remote Control)
```
public interface RemoteControl {
    void powerOn();
    void powerOff();
    void changeChannel();
    void increaseVolume();
}
```

Here:

`**RemoteControl**` tells what actions must be available

But **it does NOT tell how those actions will work**

### Different TV brands implement this interface:

ðŸ“º Samsung TV
```
public class SamsungTV implements RemoteControl {
    public void powerOn() { System.out.println("Samsung TV On"); } 
    public void powerOff() { System.out.println("Samsung TV Off"); }
    public void changeChannel() { System.out.println("Samsung Channel Changed"); }
    public void increaseVolume() { System.out.println("Samsung Volume Up"); }
}
```

ðŸ“º Sony TV
```
public class SonyTV implements RemoteControl {
    public void powerOn() { System.out.println("Sony TV On"); }
    public void powerOff() { System.out.println("Sony TV Off"); }
    public void changeChannel() { System.out.println("Sony Channel Changed"); }
    public void increaseVolume() { System.out.println("Sony Volume Up"); }
}
```

**Even though SamsungTV and SonyTV are different,
your remote works the same way:**

```

tv.powerOn();
```


Or switch to Sony:
```

```

## Definition of Interface

> Interface is a group of related methods with empty bodies

> whereas class responsibilty is to implement the  methods declared in interface

> when class agreed to implement the interface, they must provide bodies/implentation for each methods in interface
