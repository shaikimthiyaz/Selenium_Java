# TreeMap

Keys are sorted in natural order (or by a custom Comparator)

No null keys allowed

Multiple null values allowed

Slower than HashMap/LinkedHashMap due to sorting

**example:**

```
TreeMap<Integer, String> map = new TreeMap<>();
map.put(3, "Mango");
map.put(1, "Apple");
map.put(2, "Banana");
System.out.println(map);
```
output: {1=Apple, 2=Banana, 3=Mango}  // Keys are sorted

| Feature            | Description                                                                |
| ------------------ | -------------------------------------------------------------------------- |
| Implements         | `Map` and `SortedMap`                                                      |
| Order              | Keys are sorted in natural order or by custom Comparator                   |
| Null Key           | Not allowed (throws `NullPointerException`)                                |
| Null Value         | Allowed                                                                    |
| Performance        | Slower than HashMap, faster than manually sorting                          |
| Thread-Safety      | Not synchronized (use `Collections.synchronizedMap()` for thread-safe map) |
| Internal Structure | Red-Black Tree (self-balancing binary search tree)                         |


## Internal Working of TreeMap

1.Based on **Red-Black Tree**:

Self-balancing binary search tree.

Ensures that the tree is always balanced → O(log n) operations.

2.Adding a key-value pair (put):

Key is compared with root and traverses left/right.

Inserted according to key’s order.

Tree automatically balances itself if necessary.

3.Getting a value (get):

Search key like in a binary search tree → O(log n).

4. Iteration:

Keys are returned in sorted order automatically.

| Method                   | Description                |
| ------------------------ | -------------------------- |
| `put(key, value)`        | Add key-value pair         |
| `get(key)`               | Get value by key           |
| `remove(key)`            | Remove entry by key        |
| `containsKey(key)`       | Check if key exists        |
| `containsValue(value)`   | Check if value exists      |
| `size()`                 | Number of entries          |
| `firstKey()`             | Smallest key               |
| `lastKey()`              | Largest key                |
| `ceilingKey(key)`        | Smallest key ≥ given key   |
| `floorKey(key)`          | Largest key ≤ given key    |
| `subMap(fromKey, toKey)` | Returns portion of map     |
| `descendingMap()`        | Returns reverse order view |


| Feature        | HashMap    | LinkedHashMap            | TreeMap                              |
| -------------- | ---------- | ------------------------ | ------------------------------------ |
| Order          | No order   | Insertion order          | Sorted order (natural or comparator) |
| Null Key       | Yes (1)    | Yes (1)                  | No                                   |
| Null Value     | Yes        | Yes                      | Yes                                  |
| Performance    | Fastest    | Slightly slower          | Slower (O(log n))                    |
| Implementation | Hash Table | Hash Table + Linked List | Red-Black Tree                       |
