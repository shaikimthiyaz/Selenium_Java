# Arraylist

ArrayList is a dynamic (resizable) array in Java.

Part of Java Collections Framework

Package: java.util

Implements List interface

## Declaration

```
ArrayList<Integer> list = new ArrayList<>();
```

## Key Features

Dynamic size

Maintains insertion order

Allows duplicates

Allows null

Index-based (0-based)

Not synchronized (not thread-safe)

## Internal Working

Uses a normal array internally

Default capacity = 10

When full → new array created

## common methods

| Method       | Use                |
| ------------ | ------------------ |
| `add()`      | Insert element     |
| `get()`      | Access element     |
| `set()`      | Update             |
| `remove()`   | Delete             |
| `size()`     | Number of elements |
| `contains()` | Search             |
| `clear()`    | Remove all         |


## Traversing
```
for(int i=0;i<list.size();i++)
    System.out.print(list.get(i));

for(int x : list)
    System.out.print(x);
```

| Array             | ArrayList      |
| ----------------- | -------------- |
| Fixed size        | Dynamic size   |
| Stores primitives | Stores objects |
| Few methods       | Many methods   |


| ArrayList          | LinkedList         |
| ------------------ | ------------------ |
| Fast access        | Slow access        |
| Slow insert/delete | Fast insert/delete |
| Less memory        | More memory        |


### example:

```
ArrayList<String> students = new ArrayList<>();

students.add("Aman");
students.add("Riya");
students.add("Kunal");

students.remove("Riya");

System.out.println(students);
```

## Limitations

Cannot store primitive types directly
(int ❌ → Integer ✅)

Slow insertion/deletion in middle

Not thread-safe

**Best** for: Random access, read-heavy operations.
