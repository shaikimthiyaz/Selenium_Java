# What is a Map in Java?

A Map in Java is an object that stores data as **key–value pairs**, where:

Each key is **unique**

Each key maps to **exactly one value**

Values can be duplicated

**example:**

```
Map<String, String> userMap = new HashMap<>();

userMap.put("username", "john_doe");
userMap.put("email", "john@gmail.com");
userMap.put("status", "ACTIVE");

System.out.println(userMap.get("email"));
```

**output:** john@gmail.com

# Characteristics of Map

1. Map Stores Data as Key–Value Pairs

2. Keys Are Unique

3. Values Can Be Duplicate


## How Java Detects Duplicate Keys

Java checks:

**hashCode()**

**equals()**

Two keys are treated as same if:
```
key1.hashCode() == key2.hashCode()
AND
key1.equals(key2)
```

## Why Map is not part of Collection hierarchy?

Collection deals with single elements.

Map deals with pairs (key, value).

Different semantics: add() vs put().

| Collection           | Map                    |
| -------------------- | ---------------------- |
| Stores elements      | Stores key–value pairs |
| `add()`              | `put()`                |
| Indexed / sequential | Key-based lookup       |
| Single object        | Two objects per entry  |
