## Why Parallel Execution?

**Problem:** Running 500 tests sequentially takes hours.

**Solution:** Run tests in parallel threads to reduce execution time.

**Use Case:** Selenium Grid, cross-browser testing, CI/CD pipelines.

## How to Enable Parallel Execution in TestNG?

Parallel execution is controlled via testng.xml using:

parallel attribute

thread-count attribute

Options for parallel:

methods → Runs test methods in parallel.

classes → Runs classes in parallel.

tests → Runs <test> blocks in parallel.

instances → Runs multiple instances of a class in parallel.

```

<suite name="Parallel Suite" parallel="tests" thread-count="2">
    <test name="Smoke Tests">
        <classes>
            <class name="tests.LoginTests"/>
        </classes>
    </test>
    <test name="Regression Tests">
        <classes>
            <class name="tests.CheckoutTests"/>
        </classes>
    </test>
</suite>
```
